# Batch 54：App 与桌面封装产品方案

## 1. 当前可上线形态

当前产品已经可以通过浏览器访问，电脑端和手机端都可以打开。

适合阶段：

- 早期试用
- 用户反馈收集
- 样例报告展示
- AI Agent 与模板体验

## 2. 手机端方案

### 2.1 短期方案：PWA

把网页配置为 PWA，用户可以在手机上添加到主屏幕，体验接近 App。

优点：

- 快速
- 成本低
- 不需要 App Store 审核
- 适合早期免费试用

限制：

- 不是真正原生 App
- 对文件上传、后台运行、系统权限的支持有限

### 2.2 中期方案：Capacitor / React Native

当产品稳定后，可以把前端重构为标准 Web App，再用 Capacitor 或 React Native 打包成手机 App。

适合条件：

- 用户量稳定
- 核心场景稳定
- 有隐私政策、用户协议、数据删除说明
- 有明确的 App 上架目标

## 3. 电脑端方案

### 3.1 短期方案：Electron 壳

将线上网页封装为 Windows 桌面程序。

优点：

- 用户双击即可打开
- 不需要一直复制网址
- 更像正式软件

限制：

- 仍依赖线上服务
- 如果要离线，需要本地后端和本地数据库

### 3.2 中期方案：本地私有化桌面版

适合高校、企业或数据敏感用户。

需要：

- 本地数据库
- 本地文件存储
- 本地 AI 或可配置 API Key
- 安装器和卸载器

## 4. 建议路线

第一阶段：继续使用 Streamlit Cloud + 手机浏览器 + PWA 入口。

第二阶段：提供 Windows Electron 桌面壳，用户可下载安装到电脑。

第三阶段：等产品核心场景稳定后，再做 React / Next.js 前端和正式移动 App。
