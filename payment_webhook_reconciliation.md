# Batch 47 支付回调与财务对账核查清单

## 1. 资金去向

用户付款资金不会进入 Streamlit 或系统本身，而是进入你在支付平台绑定的商户账户：

- 微信支付：微信支付商户号绑定的结算账户
- 支付宝：支付宝商家账户 / 商户平台绑定的结算账户
- Stripe：Stripe Account 绑定的银行账户
- 企业转账：你指定的对公或收款账户

系统负责记录订单、验签结果、支付状态、会员开通、退款/取消订阅与对账导出。

## 2. 回调处理必须校验

每个支付平台回调都必须完成：

1. 签名校验
2. 订单号校验
3. 金额校验
4. 平台交易号保存
5. 重复回调幂等处理
6. 支付状态更新
7. 会员自动开通或续期

## 3. 异常状态

必须重点关注：

- 签名无效
- 金额不一致
- 找不到订单
- 重复回调
- 付款成功但会员未开通
- 会员开通但平台无到账记录
- 退款后会员权限未调整

## 4. 管理员后台需要定期查看

- 订单对账
- 回调事件
- 会员生命周期
- 退款/取消订阅
- 结算账户记录

## 5. 正式上线前必须完成

- 支付平台商户认证
- 结算账户绑定
- webhook 域名 HTTPS
- 回调验签测试
- 金额校验测试
- 重复回调测试
- 财务导出报表测试
- 退款流程测试
