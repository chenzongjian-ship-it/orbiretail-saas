# Batch 47 支付回调验签与财务对账指南

## 1. 核心原则

真实收费不能只依赖前端按钮、用户截图或人工口头确认。系统必须通过支付平台回调确认付款，并完成：

1. 回调验签
2. 订单号校验
3. 金额校验
4. 平台交易号保存
5. 幂等处理
6. 会员状态更新
7. 对账报表导出

## 2. 微信支付回调

微信支付 API v3 回调涉及平台证书签名校验和加密资源解密。业务落地必须验证回调来源、解密资源、校验订单号与金额，再开通会员。

需要配置：

- 微信支付商户号
- API v3 密钥
- 平台证书 / 证书自动更新机制
- 回调 URL：`/payment/webhook/wechat`

## 3. 支付宝回调

支付宝异步通知需要使用支付宝公钥对 `sign` 进行 RSA/RSA2 验签。验签通过后，还要校验：

- `out_trade_no`
- `trade_no`
- `total_amount`
- `trade_status`

回调 URL：`/payment/webhook/alipay`

## 4. Stripe Webhook

Stripe webhook 必须使用 endpoint secret 验证 `Stripe-Signature` 头。

需要配置：

- `STRIPE_WEBHOOK_SECRET`
- webhook URL：`/payment/webhook/stripe`

## 5. 结算账户绑定

Aurevia 系统本身不能替你绑定资金结算账户。真实绑定必须在支付平台完成：

- 微信支付商户平台
- 支付宝开放平台 / 商家中心
- Stripe Dashboard

系统后台只记录绑定状态、商户号、主体名称和结算账户尾号备注，用于后续核查。

## 6. 会员生命周期

系统支持：

- 支付成功自动开通会员
- 重复回调不重复开通
- 到期前提醒
- 到期自动降级为免费版
- 退款 / 取消订阅记录

## 7. 财务防纠纷清单

- 每个订单必须有内部订单号
- 每个成功付款必须有平台交易号
- 金额必须一致才开通会员
- 重复回调只记录，不重复延长
- 退款 / 取消订阅必须单独留痕
- 管理员对账表必须可导出
